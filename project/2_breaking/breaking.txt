Data Files used for Phase 2: 
(The whole dataset was downloaded as 120 separate zip files. 1 portion is simply a text file containing the data from one zip file or ~150MB.)

Pageviews-000000: First portion of the dataset
Pageviews-010000: Second portion of the dataset
Combined.txt: Contains the first and second portions stitched together
LongerCombined.txt: Contains the first four portions of the dataset



Excel Failure: Hitting the Row Cap
(see excelBreaking.png) The first and second parts of the data contain (visibly) about 1 million rows each.
Therefore, the combined part should contain at least 2 million rows, but we can see that only the first 1 million rows are loaded.
(The first two portions of the data are probably larger than the row cap, but we needed Combined.txt to definitively prove that a limit was reached)



Pandas Failure: Memory Explosion
Reading larger portions of the data using pandas.read_csv() becomes increasingly slow and memory-intensive.
We can see these effects by slowly scaling up the dataset size that pandas needs to read, as shown below:

Using 1 portion of the data (Pageviews-010000):
Wall-Clock Time: 7.79 seconds
DataFrame Memory Usage: 728.64 MB

Using 2 portions (Combined.txt):
Wall-Clock Time: 15.41 seconds
DataFrame Memory Usage: 1457.28 MB

Using 4 portions (LongerCombined.txt):
Wall-Clock Time: 30.73 seconds
DataFrame Memory Usage: 2914.56 MB

While running pandas on an even larger percentage of the dataset (300 million rows),
the wall-clock time gets even worse, and Pandas uses up almost all of the system’s RAM:
Python, because of Pandas, uses ~20k MB of RAM (see taskManager.png)
